From HTML mockup to Blogger XML in one pass

Sample docs post · BloggerEasy fixture

Product teams often design a marketing page first, then rebuild it inside Blogger by hand. This fixture shows a documentation-style layout that BloggerEasy should convert cleanly.

Tip: Prefer semantic headings and one primary CTA so palette extraction stays stable.

CLI sketch

bloggereasy gen --html data/samples/html/docs_blog.html --out out/docs_blog.xml

What to preserve

  • Title and meta description
  • Code blocks without broken entities
  • Callout borders as simple blockquotes if needed
